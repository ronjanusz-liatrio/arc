# Audit Report Template

This template defines the audit report artifact generated by `/arc-audit` at Step 5 after conducting backlog health and wave alignment checks. The report summarizes health findings across up to 15 dimensions (BH-1 through BH-5, WA-1 through WA-10), an overall health rating, recommended actions, and an optional Engineering Maturity section when Temper is installed. It is written to `docs/skill/arc/review-report.md`.

The `/arc-audit` skill reads this template and renders each `{SlotName}` placeholder with values derived from the audit session, BACKLOG.md, ROADMAP.md, VISION.md, CUSTOMER.md, and the Temper management report (when present).

---

## Template

```markdown

**Status:** {HealthRating}

- **Healthy** — No critical issues found; zero stale captured ideas; priority distribution balanced; all wave references valid; brief fields complete
- **Needs Attention** — One or more warnings present; stale ideas exist but < 5 days past threshold; minor cross-reference issues; isolated missing brief fields
- **Critical** — One or more critical issues; stale ideas > 5 days past threshold; broken wave references; bottleneck in pipeline (e.g., many shaped ideas but zero spec-ready)
```

## Field Descriptions

### GeneratedTimestamp
ISO 8601 timestamp of when the audit report was generated (e.g., `2026-04-22T14:30:00Z`).

### StalenessThresholdDays
The staleness threshold in days configured at the start of the review session (default: `14`).

### HealthRating
Overall health status: `Healthy`, `Needs Attention`, or `Critical`. Determined by the rating logic in `audit-dimensions.md`.

### BH1Finding through BH5Finding
Findings for each of the five backlog health checks. Render as a brief description of what was found or `None` if the check passed. Example: `3 ideas stale (> 14 days): "Offline Mode", "Dark Theme", "Export CSV"`.

### BH1Severity through BH5Severity
Severity level for each backlog health check: `info`, `warning`, or `critical`.

### WA1Finding through WA10Finding
Findings for each of the ten wave alignment checks. Render as a brief description or `None` / `N/A` if the check passed or was skipped.

### WA1Severity through WA10Severity
Severity level for each wave alignment check: `info`, `warning`, or `critical`.

### CapturedCount, ShapedCount, SpecReadyCount, ShippedCount, TotalCount
Counts of ideas at each lifecycle stage as of audit time. ShippedCount is derived from the wave archive at `docs/skill/arc/waves/`.

### CaptureToShapeFlow, ShapeToSpecReadyFlow, SpecReadyToShippedFlow
Prose description of pipeline health at each transition point (e.g., `"healthy flow"`, `"bottlenecked — 8 captured, 0 shaped"`).

### TemperPhase
Current Temper maturity phase from `docs/skill/temper/management-report.md`. Omit the entire Engineering Maturity section if Temper is not installed.

### TemperGatesSummary
Abbreviated gate status string using the gate-letter format (e.g., `A[P] B[P] C[F] D[W] E[P] F[N] G[W]`). `P` = pass, `F` = fail, `W` = warn, `N` = not applicable.

### TemperArtifactCoverage
Count of required Temper artifacts present vs. total required at the current phase (e.g., `2/4`).

### TemperImplication
One sentence describing what the Temper maturity state means for the product pipeline (e.g., "Foundation phase with 2 failing gates — review gate failures before planning the next wave").

### CriticalAction, WarningAction, InfoAction
Specific, actionable recommendations grouped by severity. Each action should name who should do it and how. Repeat the list item pattern for multiple actions in each group. If a group has no items, render `None`.

### StaleFixCount, TableFixCount, BriefFixCount, RoadmapRefFixCount, OtherFixDescription
Counts and descriptions of fixes applied during the interactive fixes step. If no fixes were applied, omit this section or render all counts as `0`.

## Acceptance Criteria

- The rendered audit report contains all five Backlog Health rows (BH-1 through BH-5) in the Backlog Health Findings table, each with a non-empty finding, severity, and recommended action.
- The rendered audit report contains all ten Wave Alignment rows (WA-1 through WA-10) in the Wave Alignment Findings table; rows for checks skipped due to absent artifacts render as `N/A` with an `info` severity rather than being omitted.
- The Status Distribution Summary section shows counts for all four lifecycle stages and includes a Pipeline Health assessment paragraph covering all three flow transitions.
- The Engineering Maturity section is included when Temper is installed and omitted entirely when Temper is not installed — no partial rendering of that section occurs.
- Every `{SlotName}` placeholder from this template is replaced with a concrete value in the rendered output — no literal `{SlotName}` text remains in `docs/skill/arc/review-report.md`.
