# Wave Report Template

This template defines the wave report artifact generated by `/arc-wave` at Step 10 after a delivery wave is created. The report summarizes the wave goal, selected ideas, Temper context, and handoff instructions for the SDD pipeline. It is written to `docs/skill/arc/wave-report.md`.

The `/arc-wave` skill reads this template and renders each `{SlotName}` placeholder with values derived from the wave planning session, BACKLOG.md, ROADMAP.md, and the Temper management report (when present).

---

## Template

```markdown
# Wave Report: {WaveName}

**Created:** {CreatedTimestamp}
**Theme:** {WaveTheme}
**Target:** {WaveTarget}

## Wave Goal

{WaveGoalParagraph}

## Selected Ideas

| # | Title | Priority | Summary |
|---|-------|----------|---------|
| {IdeaIndex} | [{IdeaTitle}](docs/BACKLOG.md#{IdeaAnchor}) | {IdeaPriority} | {IdeaSummary} |

## Dependencies and Blockers

- {DependencyOrBlocker}

## Engineering Readiness

- **Temper Phase:** {TemperPhase}
- **Gate Status:** {GateStatus}
- **Failing Gates:** {FailingGates}
- **Delivery Risk:** {DeliveryRisk}
- **Recommendation:** {TemperRecommendation}

## Temper Context

**Phase:** {TemperPhase}
**Hard gate failures:** {HardGateFailures}
**Scope constraint:** {ScopeConstraint}

## Handoff Instructions

For each spec-ready idea in this wave, proceed with the SDD pipeline:

1. Run `/cw-spec` with the idea's brief as input:
   - Copy the brief sections from `docs/BACKLOG.md#{IdeaAnchor}`
   - Use the brief's Problem, Proposed Solution, and Success Criteria as the spec starter prompt

2. After spec generation, continue with `/cw-plan` → `/cw-dispatch`

3. After implementation, run `/temper-audit` to evaluate gate status

### Per-Idea Handoff

| Idea | Status | Next Action |
|------|--------|-------------|
| {IdeaTitle} | spec-ready | `/cw-spec` — {IdeaHandoffSummary} |

## Backlog Status

| Status | Count |
|--------|-------|
| Captured | {CapturedCount} |
| Shaped | {ShapedCount} |
| Spec-Ready | {SpecReadyCount} |
| Shipped | {ShippedCount} |
| **Total** | **{TotalCount}** |
```

## Field Descriptions

### WaveName
The descriptive name of the delivery wave, as captured in Step 4 of `/arc-wave` (e.g., "Core Capture Flow").

### CreatedTimestamp
ISO 8601 timestamp of when the wave report was generated (e.g., `2026-04-22T14:30:00Z`).

### WaveTheme
One-sentence description of the wave's unifying theme, derived from the ideas selected.

### WaveTarget
The time estimate or milestone target for the wave. Two valid forms:
- **Populated form:** `{timeframe}` (e.g., `2 weeks`, `Q3 milestone`)
- **Placeholder form (no estimate captured):** `TBD (use /arc-wave to add)`

Never omit this line. Always render the placeholder form when no estimate was captured.

### WaveGoalParagraph
One paragraph describing what this wave achieves and why it matters. Should connect the wave to the product vision from `docs/VISION.md` when available. Not a list of features — a statement of strategic intent.

### IdeaIndex, IdeaTitle, IdeaAnchor, IdeaPriority, IdeaSummary
Repeat one table row per selected idea. `IdeaAnchor` is the lowercase-hyphenated anchor form of `IdeaTitle`.

### DependencyOrBlocker
Dependencies between selected ideas or external blockers. If none exist, render as `None identified`.

### TemperPhase
Current Temper maturity phase from `docs/skill/temper/management-report.md`. If Temper is not installed, render as `Not available`.

### GateStatus
Summary gate status (e.g., `5/7 passing`). If Temper is not installed, render as `Not available`.

### FailingGates
List of failing hard gates with brief descriptions. If none, render as `None`. If Temper is not installed, render as `Not available`.

### DeliveryRisk
Risk level — `Low`, `Medium`, or `High` — based on phase, gate status, and wave scope. If Temper is not installed, render as `Not available`.

### TemperRecommendation
One-sentence recommendation based on gate status (e.g., "Consider addressing Gate D before shipping this wave"). If Temper is not installed, render as `Not available — Temper not configured`.

### HardGateFailures
List of hard gate failures from the management report, or `None`, or `Not available` if Temper is absent.

### ScopeConstraint
How the phase and gate results constrained wave sizing and idea selection. If no constraints were applied, render as `No constraints applied`.

### IdeaHandoffSummary
Brief one-line description of the idea used as the starter prompt for `/cw-spec`.

### CapturedCount, ShapedCount, SpecReadyCount, ShippedCount, TotalCount
Snapshot counts of ideas at each lifecycle stage as of wave creation time.

## Acceptance Criteria

- The rendered wave report contains a `**Target:**` line that either states the captured timeframe verbatim or renders `TBD (use /arc-wave to add)` when no estimate was collected — the line is never omitted or left blank.
- Every `{SlotName}` placeholder from this template is replaced with a concrete value in the rendered output — no literal `{SlotName}` text remains in `docs/skill/arc/wave-report.md`.
- The rendered report contains all six top-level sections in order: Wave Goal, Selected Ideas, Dependencies and Blockers, Engineering Readiness, Temper Context, Handoff Instructions, and Backlog Status.
- The Backlog Status table shows counts for all four lifecycle stages (Captured, Shaped, Spec-Ready, Shipped) plus a bolded Total row.
- The Per-Idea Handoff table contains one row per idea selected into the wave, each with a non-empty `/cw-spec` next action summary.
