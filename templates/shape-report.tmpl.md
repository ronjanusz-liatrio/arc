# Shape Report Template

This template defines the shape report artifact generated by `/arc-shape` at Step 7 after an idea is refined from a captured stub into a shaped brief. The report records the before/after transformation, subagent analysis ratings, gaps resolved during Q&A, and open questions deferred for later. It is written to `docs/skill/arc/shape-report.md`.

The `/arc-shape` skill reads this template and renders each `{SlotName}` placeholder with values from the shaping session, including subagent findings and the final shaped brief.

---

## Field Descriptions

### IdeaTitle
The title of the idea as it appears in `docs/BACKLOG.md` (e.g., "Offline Mode Support").

### ShapedTimestamp
ISO 8601 timestamp of when the shaping session completed and the brief was written (e.g., `2026-04-22T15:00:00Z`).

### CapturedSummary
The original one-line summary from the idea's captured stub in BACKLOG.md, reproduced verbatim.

### ProblemClarityRating
Rating from the Problem Clarity subagent: `High`, `Medium`, or `Low`.

### ProblemClarityFinding
One-line key finding from the Problem Clarity subagent (e.g., `"Persona identified; impact quantified"`).

### CustomerFitRating
Rating from the Customer Fit subagent: `Strong`, `Moderate`, `Weak`, or `No fit`.

### CustomerFitFinding
One-line key finding from the Customer Fit subagent (e.g., `"Aligns with Developer persona JTBD-3"`).

### ScopeRating
Scope size rating from the Scope Boundaries subagent: `Small`, `Medium`, `Large`, or `Too large`.

### ScopeFinding
One-line key finding from the Scope Boundaries subagent (e.g., `"Single-file change; no external dependencies"`).

### FeasibilityRating
Rating from the Feasibility subagent: `Ready`, `Ready with caveats`, `Needs spike`, or `Not feasible now`.

### FeasibilityFinding
One-line key finding from the Feasibility subagent (e.g., `"Fits existing patterns; no infrastructure required"`).

### SkillDiscoveryResult
Summary of skill discovery outcome: `{N} skills found`, `No skills found`, or `Skipped`.

### SkillDiscoveryFinding
Top skill name and recommendation if skills were found (e.g., `"opentelemetry-node — install"`), `No relevant skills on skills.sh` if zero results, or `skillz plugin not installed` if skipped.

### RelevantSkillsBlock
If the feasibility subagent discovered relevant skills via `/skillz-find`, render a markdown table:

```markdown
| Skill | Author | Installs/wk | Recommendation | Relevance |
|-------|--------|-------------|----------------|-----------|
| {SkillName} | {SkillAuthor} | {SkillInstallsPerWeek} | install / investigate / avoid | {SkillRelevance} |
```

If no relevant skills were found, render:
> No relevant skills found on skills.sh for this problem domain.

If skill discovery was skipped, render:
> Skill discovery was skipped — `/skillz` plugin not installed.

### ShapedProblem
The validated problem statement from the shaped brief (1-3 sentences: who is affected, the pain point, and the impact of leaving it unsolved).

### ShapedProposedSolution
The solution description from the shaped brief (1-2 sentences: high-level approach and capability unlocked).

### ShapedSuccessCriterion
Repeat this bullet for each measurable outcome in the shaped brief's Success Criteria. Minimum 2, typically 3.

### ShapedConstraint
Repeat this bullet for each constraint. Include time, technical, team, or scope constraints.

### ShapedAssumption
Repeat this bullet for each assumption (what must be true for this solution to work).

### ShapedOpenQuestion
Repeat this bullet for each open question remaining after shaping. If all gaps were resolved, render `None`.

### ResolvedGap
One bullet per gap that was resolved during the interactive Q&A session, describing the gap and how it was resolved (e.g., `"Persona unclear → confirmed Developer persona via CUSTOMER.md review"`). If no gaps were explicitly raised, render `None`.

### DeferredOpenQuestion
One bullet per question explicitly deferred to a later phase (e.g., spec or implementation). If all questions were resolved, render `None`.

## Acceptance Criteria

- The rendered shape report contains a Subagent Analysis Summary table with all five dimension rows (Problem Clarity, Customer Fit, Scope, Feasibility, Skill Discovery), each with a non-empty rating and key finding.
- The After (Shaped Brief) section contains all six required subsections in order: Problem, Proposed Solution, Success Criteria, Constraints, Assumptions, and Open Questions — each with at least one non-placeholder value.
- The Gaps Resolved During Q&A section lists at least one entry or explicitly states `None` — it is never absent from the rendered report.
- The Open Questions Deferred section lists deferred questions or explicitly states `None` — it is never absent from the rendered report.
- Every `{SlotName}` placeholder from this template is replaced with a concrete value in the rendered output — no literal `{SlotName}` text remains in `docs/skill/arc/shape-report.md`.
