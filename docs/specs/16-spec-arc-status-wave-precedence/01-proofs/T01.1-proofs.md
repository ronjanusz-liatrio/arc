## What Changed

Step 2 now explicitly names two values to be retained for downstream steps:

- **active wave name** — verbatim string in the Wave column of the first non-Completed ROADMAP row.
- **active wave status** — verbatim string in the Status column of the same row; one of `planned` or `active` (the `Completed` value is excluded by the existing row filter).

Both values are explicitly set to `null` on the two no-wave branches:

- ROADMAP.md not found (step 2.1).
- All rows have Status `Completed` or the table is empty (step 2.4).

A new step 2.6 documents the downstream contract: Step 6 (wave-linked gap scoping) and Step 7 (precedence evaluation) consume both values, and when either is null those steps follow their no-wave branches.
