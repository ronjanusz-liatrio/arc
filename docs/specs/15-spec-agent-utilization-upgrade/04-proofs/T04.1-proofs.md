# T04.1 Proof Summary — Decide JSON Schema CLI (ajv-cli vs check-jsonschema) and document

**Task:** T04.1 (native #18) — Decide between `ajv-cli` and `check-jsonschema` for validator scripts and document the choice.  
**Completed:** 2026-04-23  
**Model:** haiku

## Decision

**Chosen:** `ajv-cli@5` + `ajv-formats@2`

**Rationale:**
- **Proven in Arc:** Already tested and used in T04.2 (schema authoring) with full success across all 4 schemas.
- **Format validation:** `ajv-formats@2` provides `format: "date-time"` validation required by backlog and wave schemas.
- **Shell integration:** npx-based installation (no global setup), clear CLI interface, proper exit codes (0=pass, 1=fail, 2+=error).
- **Lightweight footprint:** npm-native, smaller than check-jsonschema (which requires Python ecosystem).

## Canonical Invocation

All validator scripts (`scripts/validate-*.sh`) shall use:

```bash
npx --yes --package=ajv-cli@5 --package=ajv-formats@2 \
  ajv validate \
    -s schemas/<schema>.schema.json \
    -d <data>.json \
    --spec=draft7 \
    -c ajv-formats
```

Exit codes:
- `0` — validation passed
- `1` — validation failed
- `2` or higher — fatal error (schema not found, malformed JSON, etc.)
