# Questions — Round 1

## Q1: Content Scope
**Question:** What should /arc-help display?
**Answer:** Comprehensive guide — skills, workflow, artifact descriptions (VISION/CUSTOMER/ROADMAP/BACKLOG), install instructions, and README link.

## Q2: Content Source
**Question:** Should /arc-help pull content dynamically from README.md or use static content in SKILL.md?
**Answer:** Static — self-contained SKILL.md with all help text inline.

## Q3: Arguments
**Question:** Should /arc-help accept arguments to show help for a specific skill?
**Answer:** No arguments — always shows the full overview.
