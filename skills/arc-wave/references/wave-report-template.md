Format for `docs/skill/arc/wave-report.md`, generated by `/arc-wave` after wave creation.

## Template

```markdown

**Created:** {ISO 8601 timestamp}
**Theme:** {wave theme — one sentence}
**Target:** {timeframe or milestone}

## Handoff Instructions

For each spec-ready idea in this wave, proceed with the SDD pipeline:

1. Run `/cw-spec` with the idea's brief as input:
   - Copy the brief sections from `docs/BACKLOG.md#{idea-anchor}`
   - Use the brief's Problem, Proposed Solution, and Success Criteria as the spec starter prompt

2. After spec generation, continue with `/cw-plan` → `/cw-dispatch`

3. After implementation, run `/temper-audit` to evaluate gate status
```

## Field Descriptions

### Target
The `**Target:**` header line has two valid rendered forms:

- **Populated form** — when a time estimate was captured, render verbatim:
  ```markdown
  **Target:** {timeframe or milestone}
  ```
  For example: `**Target:** 2 weeks` or `**Target:** Q3 milestone`.
- **Placeholder form** — when no time estimate was captured at wave creation, render the literal line:
  ```markdown
  **Target:** TBD (use /arc-wave to add)
  ```

The placeholder string is used verbatim (not `TBD`, not `TBD (...)` with a different hint). It matches the form used in the ROADMAP wave entry generated by Step 6 of `skills/arc-wave/SKILL.md`. The ROADMAP summary table uses the shorter `TBD` value in its `Target` column, but the wave report header always uses the full placeholder form so the reader sees the same reminder hint in both places.

Existing wave reports with a populated Target value are never rewritten or normalized — the placeholder only applies to newly created waves that skip the estimate.

### Wave Goal
Should connect the wave to the product vision. Not a list of features — a statement of strategic intent.

### Temper Context
Read from `docs/management-report.md` if present. The three fields capture:
- **Phase** — Current Temper maturity phase (Spike through Maturity)
- **Hard gate failures** — Any gates that failed in the most recent evaluation
- **Scope constraint** — How the phase and gate results influenced wave sizing and idea selection

### Handoff Instructions
The per-idea handoff table provides a ready-to-use starter prompt for each `/cw-spec` invocation. This bridges the gap between Arc's brief and claude-workflow's spec.

### Backlog Status
Snapshot of the backlog at wave creation time. Provides context for stakeholders on pipeline health.
